## Rig Error (Invalid Avatar – Hips / LeftUpperLeg 없음)

**원인:**  
어떤 모델의 **Animator**가 **Humanoid** 아바타를 쓰도록 되어 있는데, 해당 FBX/리그에 Humanoid 필수 본(Hips, LeftUpperLeg 등)이 없을 때 발생합니다.

**해결 (Unity에서 수동):**

1. 콘솔에서 해당 Rig 에러가 나올 때 **에러 메시지 더블클릭**해 해당 오브젝트/에셋으로 이동하거나,
2. **Player** 또는 캐릭터가 들어 있는 **프리팹**을 연 뒤,
3. **루트 오브젝트** 선택 → **Animator** 컴포넌트 확인,
4. 사용 중인 **Avatar**가 있는 **FBX**를 Project에서 선택,
5. Inspector **Rig** 탭에서 **Animation Type**을 **Humanoid** → **Generic** 으로 바꾼 뒤 **Apply**.

또는 해당 캐릭터가 **Humanoid 애니메이션**을 쓰지 않는다면, Animator에서 **Avatar**를 None으로 두고 **Apply** 해도 됩니다.

## Mesh / FBX 관련 경고 (self-intersecting, no normals, tangents)

- **"A polygon of Mesh 'XXX' is self-intersecting and has been discarded"**  
  해당 메쉬의 폴리곤이 자기 교차(self-intersecting)라서 Unity가 일부 폴리곤을 버린 것입니다.  
  → 3D 툴(Blender, Maya 등)에서 메쉬 수정 후 재 export 하거나, 시각적으로 문제 없으면 무시해도 됨.

- **"Mesh 'XXX' has no normals. Recalculating normals"**  
  Unity가 자동으로 노멀을 다시 계산한 것입니다.  
  → 에셋 스토어/외부 FBX에서 자주 나오며, 보통 무시해도 됨.

- **"Can't import tangents and binormals, because mesh 'XXX' doesn't have any"**  
  메쉬에 탄젠트/바이노멀이 없어서 나오는 Import 경고입니다.  
  → 시각/라이트맵에 큰 문제 없으면 무시 가능.

위 항목들은 **코드로 일괄 수정하기 어렵고**, 필요 시 해당 FBX만 3D 툴에서 수정 후 재 import 하는 방식으로 처리하면 됩니다.
