# Part 2 — 게임의 뼈대 설계 (6각 월드)

지금 만들려는 건:

- 6각 타일
- 중심 홈
- 외부 확장형
- 수집/성장 기반

그럼 "예쁘게"가 아니라 **확장 가능하고 안 터지는 구조**로 설계해야 한다.

---

## 🎯 0️⃣ 먼저 결정해야 할 것

6각에는 2가지 방식이 있다:

- **Flat Top** (위아래가 평평)
- **Pointy Top** (좌우가 평평)

Flat Top 사용 중이면 그 기준으로 설계한다.

---

## 1️⃣ 좌표 시스템을 먼저 고정해야 한다

월드 구조 설계의 80%는 좌표에서 끝난다.

6각 타일은 보통:

### ✅ Axial Coordinates (추천)

```
(q, r)
```

### 또는 Cube Coordinates

```
(x, y, z)  // x+y+z=0
```

---

### 🎯 추천: Axial (q, r)

이유:

- 구현 단순
- 거리 계산 쉬움
- 링 생성 쉬움
- 무한 확장 가능

---

## 2️⃣ 월드 구조 설계

구조:

- 중심 = r=0
- 첫 링 = 6개
- 둘째 링 = 12개
- 셋째 링 = 18개
- …

**총 타일 수 공식:**

```
1 + 3R(R+1)
```

| R | 타일 수 |
|---|--------|
| 3 | 37     |
| 4 | 61     |
| 5 | 91     |
| 6 | 127    |
| 7 | 169    |
| 8 | 217    |

---

## 🎯 추천 구조 (안전한 모바일 볼륨)

### R = 6 (127타일)

- 홈 1
- 6 + 12 + 18 + 24 + 30 + 36

**총 127** — "월드 1개"로 딱 좋다.

---

## 3️⃣ 타일 데이터 구조 설계 (중요)

타일은 **GameObject가 아니라 데이터 기반**이어야 한다.

### TileData 예시

```csharp
public class TileData
{
    public Vector2Int axial;
    public int ringLevel;
    public TileType type;
    public bool unlocked;
}
```

**GameObject는 단순 "뷰"일 뿐이다.**

---

## 4️⃣ 월드 생성 방식

### 🔥 추천: 중앙에서 링 단위 생성

```csharp
for (int q = -R; q <= R; q++)
{
    for (int r = -R; r <= R; r++)
    {
        if (Mathf.Abs(q + r) <= R)
        {
            CreateTile(q, r);
        }
    }
}
```

이 조건이 육각 범위다.

---

## 5️⃣ 확장 구조 설계

### 단계 기반 오픈 방식

- R=2까지 기본 오픈
- 자원 모이면 R=3 오픈
- 그 다음 R=4 오픈

즉, **currentMaxRing** 변수 하나로 관리.

---

## 6️⃣ 자원 배치 설계

링 기반 티어 추천:

| Ring | 자원   |
|------|--------|
| 0    | 홈     |
| 1    | 나무   |
| 2    | 돌     |
| 3    | 철     |
| 4    | 금     |
| 5    | 수정   |
| 6    | 희귀   |

이렇게 가면 자연스럽다.

---

## 7️⃣ 오브젝트 배치 구조

- ❌ 타일마다 랜덤 배치 X
- ✅ **TileType 기반 배치**

예:

```csharp
switch (tile.type)
{
    case TileType.Forest:
        SpawnTree();
        break;
}
```

---

## 8️⃣ 성능 안전 설계

127 타일 기준:

- 타일 메쉬 1개 (공유)
- 환경 머티리얼 1개
- 오브젝트 풀링 사용
- 멀리 있는 타일 비활성화 가능

---

## 9️⃣ 카메라 설계

세로 모드 기준:

- 화면에 보이는 링은 최대 3~4
- 나머지는 미리 생성하되 비활성화 가능

---

## 🔥 가장 중요한 설계 원칙

> **타일 = 데이터**  
> **오브젝트 = 표현**  
> **확장 = 숫자 변경**

이 구조로 만들면 무한 확장도 가능하다.

---

## 📌 확장 전략

### 소프트 캡 방식

- 실제 생성 R=6
- 외곽 링은 테마 교체
- 타일 재활용

이게 모바일에서 가장 안전하다.

---

3. 플레이어 이동

5. 자원 수집 루프

# 1️⃣ 먼저 오해부터 풀자

데이터 기반이라고 해서

> ❌ 씬에서 테스트하면 안 된다
> ❌ 눈으로 배치하면 안 된다

이게 아니다.

정답은:

> ✅ 씬에서 ‘설계’하고
> ✅ 그 결과를 ‘데이터로 저장’한다

이다.

---

# 2️⃣ 복잡한 요소들 어떻게 데이터로 관리하냐?

너가 말한 요소들 하나씩 보자.

---

## 🔹 1. 타일마다 높낮이 다름

이건 단순하다.

```csharp
public float height;
```

데이터에 숫자로 저장하면 끝이다.

스폰할 때:

```csharp
tileObject.transform.localScale = new Vector3(1, height, 1);
```

또는

```csharp
tileObject.transform.position += Vector3.up * heightOffset;
```

---

## 🔹 2. 타일마다 색 다름

```csharp
public Color tileColor;
```

스폰할 때 MaterialPropertyBlock으로 적용.

---

## 🔹 3. 타일마다 모양 다름

모양은 보통:

* 기본 타일
* 절벽 타일
* 물 타일
* 모래 타일

이런 프리팹 차이다.

```csharp
public TileVisualType visualType;
```

그리고

```csharp
SpawnPrefab(tile.visualType);
```

---

## 🔹 4. 해금칸 표시

이건 데이터가 이렇게 된다:

```csharp
public bool unlocked;
public int unlockCost;
```

그리고 로직:

* 인접한 타일 중 unlocked == true 있으면
* Unlock 버튼 표시

---

## 🔹 5. 타일 위 오브젝트 배치

이건 이렇게 한다:

```csharp
public List<ObjectSpawnData> objects;
```

예:

```csharp
[System.Serializable]
public class ObjectSpawnData
{
    public ObjectType type;
    public Vector3 localOffset;
}
```

스폰 시:

```csharp
SpawnObject(tileData.objects[i]);
```
